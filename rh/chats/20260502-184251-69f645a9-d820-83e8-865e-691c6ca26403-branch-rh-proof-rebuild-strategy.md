## What the rebuild changed

The rebuild made several simplifying choices.

### 1. It stopped trying to classify all finite cores

V1’s three-point branch appears because V1 says: suppose a finite bad core exists; then classify its minimal affine dependence. That naturally forces the \(k=2,3,4\) trichotomy.

The rebuild is not currently doing that. It is not saying:

\[
\text{bad finite core}\Rightarrow\text{affine dependence}\Rightarrow k=2,3,4.
\]

Instead, it is trying to build a direct local obstruction from jet–Gram blocks.

That is a different proof spine.

### 2. It chose a scalar two-point channel

The rebuild’s completed projection is on a single \(2\times2\) whitened block:

\[
\widehat\Omega(s;m)\in M_2(\mathbb R),
\]

with scalar channel

\[
\mathcal A_2(X)=X_{12}+X_{21}.
\]

This is not the same object as V1’s lifted curve \(\Xi(h)\), \(Q_v(h)\), or finite-core affine-dependence package.

So Section 6 does not need a three-point projection because Section 6 is no longer a finite-core incidence classifier. It is a scalar projection section.

### 3. It quarantined V1’s calibration machinery

V1 used matrix-level calibration objects such as \(M(d)\), \(A_{\val}\), and \(\Psi_{x,d}\). The rebuild has intentionally not imported that machinery. The new toy phase was also chosen to support the scalar channel, not to preserve V1’s old \(M(d)\)-level calibration.

That means any V1 theorem whose proof depends on \(M(d)\), \(A_{\val}\), \(\Psi_{x,d}\), lifted finite-core coordinates, or three-point collinearity is **not automatically available** in the rebuild.

### 4. It treats four-point as the separate full-quartet frontier

Four-point remains important because it corresponds to the full symmetry quartet

\[
\rho,\quad \bar\rho,\quad 1-\rho,\quad 1-\bar\rho,
\]

and also because V1’s finite-core geometry reaches its natural affine-circuit endpoint at \(k=4\). The original paper says minimal defect-free bad cores have size at most four, with \(k=4\) as the four-point affine circuit. 

So the rebuild keeps four-point as a real frontier.

But it does **not** include three-point because three-point is neither:

1. the scalar two-center branch, nor  
2. the full quartet/four-center branch.

## Why V1 needed three-point but the rebuild does not, conditionally

V1 needed three-point because V1’s reduction produced arbitrary minimal finite cores. Once arbitrary finite cores are in play, a three-point collinearity case is unavoidable.

The rebuild avoids three-point only by **not using that reduction**.

Its intended branch logic is:

\[
\text{off-line packet}
\Rightarrow
\begin{cases}
\text{two-point scalar obstruction},\\
\text{or four-point/full-quartet obstruction}.
\end{cases}
\]

If that branch-exhaustion statement is proved, then no separate three-point branch is needed.

But if branch exhaustion is not proved, a referee can ask:

> Why could an off-line packet not fall into a V1-style three-point lifted collinearity regime that is invisible to the two-point scalar channel and not yet captured by the four-point branch?

That is the important remaining logical checkpoint.
