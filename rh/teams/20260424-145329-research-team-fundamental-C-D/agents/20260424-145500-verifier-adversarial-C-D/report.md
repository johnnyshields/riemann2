### 2. `gap-closer-C-diagonal-merger/report.md`

The key issue is that the report treats C1 and C3 as mostly formal once the corrected package is fixed, but that is itself conditional on no additional hidden gauge/state surviving the reduction map `\mathcal R`. The involutive-branch support cited in the report is too narrow to justify that globally. So the trustworthy output is weaker:

- C2 is definitely still open and is the main blocker.
- C1/C3 may be easier than C2, but they are not yet discharged, and calling them “mostly formalization” is not proved from the current draft alone.

Hence the report reduces C but does not close it, and its “only genuinely hard item is C2” claim should remain conditional.

### 3. `gap-closer-D-kerphik/report.md`

This report makes a useful patchwise algebraic observation but does not prove D. On the patch `\{c\neq 0, v_5\neq 0\}` it shows that, after fixing
\[
\widehat\Psi=(Y,x,\Delta),

### 4. `gap-closer-UV002-contradiction/report.md`

The report’s own strongest objection already points to the problem: item 3 is not mere formatting. But it does not identify the actual missing package/provenance theorem, only the region where it must live. So it remains heuristic support and cannot promote any part of C or D.
