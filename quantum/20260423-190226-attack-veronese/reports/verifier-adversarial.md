# verifier-adversarial

- **Claim**: The Veronese-family generalization is mathematically real, but should be promoted carefully. At most it belongs in the note as a benchmark-family theorem illustrating the ambient-subspace mechanism; it should not become part of the main theorem spine.
- **Status**: proved
- **Evidence**: Proved: the current note already has the safe general theorem at the right level of abstraction, namely that ambient osculating subspaces `A_r` are gauge- and reparameterization-invariant and therefore yield canonical transport-free principal-angle data; the quartit and qutrit formulas are presented as benchmark examples inside that framework. Proved: the deposited Veronese theorem does package those low-dimensional formulas into a clean family statement at order `n-1`, with an explicit normal-vector formula and principal-angle cosine formula. Conditional: this family theorem can improve the examples section by showing the polynomial benchmarks are not isolated accidents. Missing: any argument that it changes the main honest message of the note, or that it advances the unresolved matrix/covariant-frame issues. So it should be used as a unifying benchmark theorem, not as a new central claim.
- **Exact refs**: `quantum/paper/jet_gram_quantum_note.md:264-370`; `quantum/findings.md:50-64`; `quantum/20260423-190226-attack-veronese/reports/gap-veronese-theorem.md:3-23`; `quantum/20260423-190226-attack-veronese/notes/coordinator-brief.md:3-13`.
- **Dependencies**: The already-safe ambient-osculating-subspace theorem; the deposited Veronese theorem itself; the existing qutrit/quartit benchmark material.
- **Strongest objection**: Without careful framing, the Veronese theorem can look like ornament: a neat polynomial family that does not by itself resolve the central matrix-versus-subspace issue. Promoted too hard, it risks confusing a benchmark-family unification with a broader theorem about generic quantum families.
- **Needed for promotion**: If added to the note, keep it short and clearly labeled as a benchmark-family theorem. Do not use it to imply more than it proves.
