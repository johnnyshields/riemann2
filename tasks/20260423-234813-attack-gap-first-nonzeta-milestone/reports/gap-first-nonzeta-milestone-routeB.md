## Claim

The proposed first non-zeta milestone is still too large if it is stated as the paired source-plus-slot realization theorem. From current scope alone, that target still hides at least two theorem burdens under one label: upstairs paired exact-source realization and downstairs exact local-slot identification. The smallest milestone that is still nontrivial and honest is narrower: the corrected-whitening transport bundle, namely denominator comparability plus omitted-smooth holomorphy, holomorphic corrected whitening with positive same-point blocks, and preserved post-`\Phi_K` transfer scale. That is a real family-level theorem package, but it does not pretend to realize the exact manuscript `S(m)` slot or any endgame consequence.

To keep the roadmap honest, two statements should be separated.

First: `best first non-zeta exact-source realization target` remains the primitive Dirichlet paired theorem, with scoped wording.

Second: `smallest honest non-zeta milestone` is the corrected-whitening transport bundle, because it has a cleaner boundary and explicitly defers exact slot realization, remainder dominance, odd/transverse realization, and boundary domination.

## Evidence

### Proved

The supplied notes already separate the burdens in a way that blocks calling paired source-plus-slot realization a small milestone. `paired_slot_realization.md` states that the target is best phrased as two linked theorem burdens: upstairs paired source realization and downstairs exact local-slot identification. That is already larger than a single irreducible theorem-sized checkpoint.

The same note also states that even after exact paired slot realization, the following still remain: remainder dominance, corrected odd/transverse realization, denominator/whitening control, boundary estimates, and calibration uniformity issues. So the proposed milestone does not merely prove a non-zeta realization fact; it still sits strictly before several independent theorem-sized packages.

By contrast, `corrected_whitening_transport.md` isolates a bounded intermediate bundle with exactly three parts: denominator comparability plus omitted-smooth holomorphy, holomorphic corrected whitening with positive same-point blocks, and preserved post-`\Phi_K` transfer scale. The note is explicit that this is only a partial intermediate bundle. That limitation is a feature here: it keeps the milestone honest.

`aval_transport.md` reinforces the same boundary. It says `A_val` transport is only a first-order local algebra statement and does not include the full `\Delta = S A_{\val} + R` theorem in a new family, calibration, remainder dominance, or odd/transverse consequences. So any milestone that advertises more than the corrected-whitening layer is already carrying theorem debt beyond local transport.

`boundary_package.md` sharpens the later cutoff. Even after corrected whitening, the next standalone target is not the bare boundary theorem but the broader corrected transverse realization-and-boundary package. This means the roadmap after the smaller milestone is still nontrivial, but it is honestly staged.

### Conditional on scoped interpretation

If the phrase `first non-zeta milestone` is meant to mean `first non-zeta exact-source realization target`, then the priority note and paired-source notes still support the paired Dirichlet route as the cleanest candidate from current scope alone. But that statement is only safe with the exact-source qualifier. It is not the same as saying this is the smallest or cleanest milestone overall.

Conditional on proving the paired source-plus-slot theorem, the manuscript's leading-channel interface is already in place, and some upstairs positivity/kernel structure becomes automatic. But those conditional gains begin only after exact slot identification, and they still leave remainder dominance, odd/transverse realization, denominator/whitening control, boundary estimates, and variable-`x` uniformity as separate tasks.

### Missing

Missing from the proposed paired source-plus-slot milestone, if it is advertised as a true early milestone, is a justification that the two linked burdens really behave like one theorem rather than two substantial theorems with different inputs.

Missing is any argument that exact local-slot identification is minor bookkeeping. The supplied notes repeatedly treat it as theorem-sized.

Missing is any reason to treat remainder dominance, odd/transverse realization, boundary control, or variable-`x` calibration as downstream formalities. The notes isolate them as separate burdens.

Therefore the honest staging is:

1. corrected-whitening transport bundle;
2. exact paired source realization plus exact local-slot identification, if the goal is the first exact-source milestone;
3. remainder dominance for the calibrated value channel;
4. corrected transverse realization-and-boundary package;
5. any remaining calibration-uniformity issues.
